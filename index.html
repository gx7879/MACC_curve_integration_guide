1 架構與認證

前端不直接呼叫 aiportal-svc，而是呼叫 BFF；BFF 以 1:1 代理轉發至服務（無 data 外層包裝 — svc 的 JSON 原樣回傳）。

  瀏覽器 ──HTTP──▶  aiportal-bff                    ──HTTP──▶  aiportal-svc
                     /api/aip-web/macc/constants               /api/macc/constants    (型錄 + 地區常數)
                     /api/aip-web/macc/preview                 /api/macc/preview      (各措施 MAC，未排序)
                     /api/aip-web/macc/calculate               /api/macc/calculate    (已排序曲線 + 彙總)
                     /api/aip-web/macc/records[/{sn}]          /api/macc/records[...]  (儲存 / 歷史，使用者範圍)
認證 — 每次呼叫都帶上使用者的 Keycloak Authorization: Bearer <jwt>。
  • /records* 與 /calculate：Bearer 會被轉發；/records* 為使用者範圍（解析 CurrentUserEmail）。
  • /preview：會轉發 Bearer，但非使用者範圍（無害）。
  • /constants：非使用者範圍；無 Bearer 時 BFF 退回服務 token。
快取與多語
  • GET /constants 在 BFF 端快取 1800 秒。/calculate 與 /preview 不快取（依輸入而變的 POST）。
  • 每個端點皆接受 ?lang=（預設 en，或 zh-TW），由後端在地化地區／類別／設備／措施標籤。

2 預期使用者流程

四個階段，對應設計板上的區段標題：功能入口 → 填寫數據 → 分析結果，並以 Navi × ESM 型錄 作為背後的參考資料。

1 · 功能入口

入口

Feature entry / landing
  • 使用者進入 減碳效益評估（Sustaint Navi 下），點擊 立即評估。
  • 預先載入型錄以填充地區下拉選單與預設輸入值。
GET /constants
2 · 填寫數據

填寫數據

輸入評估情境
  • 選地區；設定年用電量、電費、碳費／ICP、減碳目標。
  • 設定系統用電佔比（category_ratios，總和須 ≤ 100%）。
  • 對所有措施評分以繪製選用卡片，附 MAC 數值與「最佳」徽章。
POST /preview
3 · 分析結果

分析結果

MACC 曲線
  • 送出已選措施 → 繪製曲線 + 結果表 + 彙總。
  • 狀態：正常 · Hover · Error · Blank · 下拉收合。
  • 可選擇將命名情境存檔。
POST /calculate · POST /records
★ 參考資料

型錄

Navi × ESM 與常數
  • 5 大類別共 17 項 ESM 措施。
  • 各國常數：碳費、電網排放係數、電價、幣別、匯率。
GET /constants
為何同時需要 /preview 與 /calculate？ 措施的 MAC 取決於使用者輸入（用電量、佔比、地區），無法由靜態型錄取得。/preview 在目前輸入下對每一項措施評分（型錄順序、未排序），用以驅動選用卡片；/calculate 取已選子集，建立已排序的累積曲線。選用畫面上輸入變動時呼叫 /preview；使用者送出時呼叫 /calculate。

3 API 端點參考

所有路徑相對於 BFF 來源，並以 /api 為前綴。

方法路徑用途認證快取
GET/api/aip-web/macc/constants地區常數 + ESM 型錄 + 表單預設值選用1800 秒

共用請求型別

欄位型別必填說明
regionstring是地區代碼：TW CN JP SG KR MY TH VN
total_kwhnumber是年度用電量（kWh）
electricity_ratenumber否當地幣別 / kWh。省略 → 地區 electricity_price.avg

欄位型別適用說明

load_ratenumber?特殊

5 POST /preview — 選用設備卡片

在 填寫數據 畫面，每張措施卡片顯示其在目前輸入下的 MAC，且每個類別以「最佳」徽章標示 MAC 最低的選項。當地區、total_kwh、electricity_rate 或 category_ratios 變動時重新呼叫。

7 紀錄 — 儲存與歷史

使用者範圍的情境保存（左側邊欄歷史）。POST /records 接受與 /calculate 完全相同的內容再加上 name；後端會重新執行計算並同時保存輸入與結果。

儲存

POST /api/aip-web/macc/records
{ "name": "TW baseline 2026",
  "region": "TW", "total_kwh": 25000000,
  "reduction_target_tco2e": 355,
  "category_ratios": { "空調":0.5,"空壓":0.2,"照明":0.2 },
  "selected_measures": [ { "measure_id":"DELTA_HV0005" } ] }

列表 / 取得

GET /api/aip-web/macc/records
// → [{ sn, name, region, created_at, updated_at }]

GET /api/aip-web/macc/records/{sn}
// → { sn, name, region, inputs, result, created_at, updated_at }
這三個端點需要真實的使用者 Bearer（會解析 CurrentUserEmail）。此處出現 401 表示 JWT 遺失或過期 — 重新認證後再試。

8 繪製曲線與其狀態

座標軸與長條

  • X 軸＝累積年度 CO₂ 減量（tCO₂e）。每根長條由 x_start 畫到 x_end，寬度即 annual_reduction_tco2e。
  • Y 軸＝MAC（當地幣別 / tCO₂e）。長條高度＝mac（可為負 → 位於零線下方）。
  • 長條已排序（成本最低者在前）且連續 — 依陣列順序繪製，前端不需再排序。

參考線

  • 紅色虛線（垂直）位於 reference_lines.reduction_target_tco2e（步驟一目標）。
  • 水平線位於 reference_lines.carbon_price（碳費 / ICP）。
  • 標頭 KPI 取自 totals（target_achievement_pct、total_annual_saving…）。
成本最低 有利可圖 低成本 邊際 需補貼

五種畫面狀態

狀態觸發顯示內容
正常/calculate 成功且有長條曲線 + 結果表 + 彙總 KPI。
Hover指標移至某長條上提示：標題、MAC、kwh_saved、annual_saving、annual_reduction_tco2e。
Blank尚未選任何措施／首次執行前空白曲線框 + 引導填寫數據。
ErrorHTTP 4xx/5xx（見下）行內錯誤；保留輸入以便重試。
收合下拉／面板收合純版面狀態 — 不需額外呼叫。

錯誤對應

  • 400 — 輸入錯誤：未知的 region、未知的 measure_id，或 category_ratios 總和 > 1.0。請顯示 detail 訊息。
  • 401 — 使用者範圍路由缺少／過期的 Bearer。重新認證。
  • 404 — /constants?region= 未知地區，或未知的紀錄 sn。
  • 502 — BFF 無法連線至 svc。提供重試。

10 引擎公式 （讓欄位更易理解）

前端不需計算這些 — 但理解公式有助於釐清各措施類型仰賴哪些輸入。所有中間運算以 USD 進行，再透過 fx 將 MAC 與年節省金額換算為當地幣別。不做折現（型錄中的 discount_rate 僅為參考資料）。

節省能源（kWh/年）— 依 algorithm 分流

general = total_kwh × category_ratio × device_category_share × saving_rate
special = rated_power_kw × quantity × load_rate × annual_hours × saving_rate

接著，每項措施

annual_saving_local      = kwh_saved × electricity_rate_local
annual_reduction_tco2e   = kwh_saved × grid_emission_factor ÷ 1000        // kg → 公噸
MAC_usd                  = (investment_cost_usd ÷ lifecycle_years − annual_saving_usd) ÷ annual_reduction
MAC_local                = MAC_usd × fx
MAC 的正負號：回收年限 = 生命週期 → MAC 為 0；回收年限 < 生命週期 → 負值（有利可圖，長條落在零線下方）；回收年限 > 生命週期 → 正值（需補貼）。若 annual_reduction 為 0（佔比／節能率為零），MAC 回報為 0 而非除以零。

11 注意事項與待辦

  • category_ratios 總和須 ≤ 1.0 — 否則 /calculate 與 /preview 回 400。送出前於表單驗證。鍵值為中文類別名稱（空調 空壓 照明 電力 生產），不隨 lang 改變。
  • 特殊 vs 一般輸入 — 僅電力/生產措施會讀取 rated_power_kw / quantity / load_rate / annual_hours。一般措施只送 measure_id；多餘數值欄位會被忽略。
  • 類別佔比不適用於特殊措施 — 其節省源自設備規格，而非 total_kwh × ratio。只需為實際使用的一般類別提供佔比。
  • preview 刻意未排序 — 於前端排序並挑出「最佳」（各類別 MAC 最小者）。
  • calculate 已預先排序 — 勿重新排序；直接用 x_start/x_end 依陣列順序繪製長條。
  • 數值為佔位值 — fx、最終 ESM 數字與投資成本縮放仍待 #1450（確認常數）／#1451（EnergiQ 同步）。量級會變動，但 API 形狀不變。前端勿硬編任何型錄值 — 一律從 /constants 讀取。
  • 幣別 — 所有金額欄位（MAC、節省、碳價）皆以 response.currency 計（地區碳費幣別，於現行資料中等同電價幣別）。
  • 快取 — /constants 在 BFF 端快取 1800 秒；若地區／措施有編輯上線，須等待最多約 30 分鐘或主動清除快取鍵。
